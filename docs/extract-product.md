# `extract-product` — парсинг товара по ссылке

BUILD-слой для Quotcat: по ссылке на страницу товара движок возвращает
**нейтральный структурированный товар**. Граница: движок отдаёт **ЧТО** (поля
товара), а как это ляжет в offer и по какой цене для дилерского профиля — решает
Quotcat. Цена со страницы — лишь одно из извлечённых полей.

- Эндпоинт: `POST /api/v1/extract-product`
- Scope: `extract`, **только секретный ключ** (`sk_…`) — доверенный server-to-server вызов.
- Код: [`src/routes/extract-product.ts`](../src/routes/extract-product.ts), [`src/lib/engine/extractProduct.ts`](../src/lib/engine/extractProduct.ts), [`src/lib/engine/pageFetch.ts`](../src/lib/engine/pageFetch.ts).

## Запрос / ответ

```http
POST /api/v1/extract-product
X-API-Key: sk_...
Content-Type: application/json

{ "url": "https://shop.example/product/123" }
```

Успех — конверт `{ data, meta }`:
```json
{
  "data": {
    "name": "TEXA Navigator",
    "price": 154900,
    "currency": "RUB",
    "sku": "TXB-001",
    "specs": { "Гарантия": "12 мес", "Вес": "2 кг" },
    "images": ["https://shop.example/i/1.jpg"],
    "description": "Краткое описание",
    "sourceUrl": "https://shop.example/product/123"
  },
  "meta": {
    "source": "llm",            // structured | llm | cache
    "models": ["yandexgpt-lite"],
    "usageTokens": 2730,         // суммарные токены (промпт+ответ); 0 для structured/cache
    "costRub": 0.546             // оценка стоимости операции
  }
}
```

Ошибка — конверт `{ error: { code, message } }`.

## Конвейер (что происходит по шагам)

```
url
 │
 ├─ 1. КЭШ по url ───────────────► hit → отдать (source=cache, 0 ₽)
 │
 ├─ 2. fetchPage: SSRF-проверка (isSafeUrl) → https-only → лимит 5 МБ / таймаут 10с
 │      → html→text + ld+json + OpenGraph
 │
 ├─ 3. «тонкая» страница (text<200 && нет ld+json)?
 │      └─ ENABLE_HEADLESS=1 → рендер Playwright и повтор; всё ещё тонкая → 422 EAI_EMPTY_PAGE
 │
 ├─ 4. extractProduct:
 │      a. structured-first: узел Product из ld+json + OpenGraph → детерминированные поля
 │      b. (опц.) структура «полная» и EXTRACT_SKIP_LLM_WHEN_COMPLETE=1 → БЕЗ LLM (source=structured)
 │      c. иначе LLM (lite) с фокусной подсказкой (узел Product) + текст (EXTRACT_TEXT_CHARS)
 │      d. каскад: lite не дал даже названия на непустой странице → повтор на Pro
 │      e. finalize: структура ПЕРЕТИРАЕТ LLM (точность), specs мерджатся, sourceUrl со страницы
 │
 └─ 5. записать в кэш → вернуть { data, meta }
```

## Оптимизации стоимости (без потери качества)

| # | Что | Как настроить | Эффект |
|---|---|---|---|
| 1 | **structured-first** — поля из `ld+json`/OpenGraph берутся детерминированно и перетирают LLM | всегда вкл; полный пропуск LLM — `EXTRACT_SKIP_LLM_WHEN_COMPLETE=1` (по умолч. выкл, чтобы не терять specs/description) | точность ↑; при пропуске — операция 0 ₽ |
| 2 | **фокусная подсказка** — в LLM уходит только узел Product, не все ld+json блоки; текст обрезается | `EXTRACT_TEXT_CHARS` (дефолт 6000) | меньше токенов |
| 3 | **каскад lite→pro** — Pro только если lite не вытащил название на непустой странице | `YANDEX_CHAT_MODEL` задаёт базовую модель | качество ≈ Pro, средняя цена ≈ Lite |
| 4 | **кэш по url** — повтор без fetch и без LLM | `EXTRACT_CACHE_TTL_MS` (дефолт 1ч), `EXTRACT_CACHE_MAX` (500); 0 = выкл | повторы = 0 ₽ |
| 5 | **usage + расчёт ₽** — реальные токены из ответа провайдера | тарифы `YANDEX_LITE_RUB_PER_1K` (0.20), `YANDEX_PRO_RUB_PER_1K` (1.20) — **уточнить в биллинге** | себестоимость в `meta.costRub` |

> Яндекс тарифицирует **промпт + ответ суммарно**. `costRub = Σ (totalTokens/1000 × тариф_модели)` по всем вызовам (каскад считается корректно: lite-токены по lite-тарифу + pro по pro).

## Переменные окружения

| Переменная | Дефолт | Назначение |
|---|---|---|
| `ENABLE_HEADLESS` | `0` | Playwright-рендер для JS/SPA-страниц (Фаза 2). Прод: `npx playwright install chromium` |
| `EXTRACT_SKIP_LLM_WHEN_COMPLETE` | `0` | Полный пропуск LLM, если структура полная (name+price+description+images) |
| `EXTRACT_TEXT_CHARS` | `6000` | Сколько символов текста страницы отдавать в LLM |
| `EXTRACT_CACHE_TTL_MS` | `3600000` | TTL кэша результатов; `0` — выключить |
| `EXTRACT_CACHE_MAX` | `500` | Максимум записей в кэше |
| `YANDEX_LITE_RUB_PER_1K` / `YANDEX_PRO_RUB_PER_1K` | `0.20` / `1.20` | Тарифы для оценки стоимости (ориентир) |
| `YANDEX_CHAT_MODEL` | `yandexgpt-lite` | Базовая модель экстракции |

## Коды ошибок

| Код | HTTP | Когда |
|---|---|---|
| `EAI_FORBIDDEN` | 403 | ключ не секретный |
| `EAI_RATE_LIMITED` | 429 | превышен лимит запросов на ключ |
| `EAI_BAD_REQUEST` | 400 | не передан `url` |
| `EAI_INVALID_URL` / `EAI_HTTPS_ONLY` / `EAI_UNSAFE_URL` | 400 | плохой/не-https/SSRF-небезопасный URL |
| `EAI_FETCH_FAILED` / `EAI_NOT_HTML` / `EAI_NO_BODY` | 400 | страница не загрузилась / не HTML |
| `EAI_TOO_LARGE` | 413 | страница > 5 МБ |
| `EAI_EMPTY_PAGE` | 422 | пустой каркас (JS-рендеринг); включите `ENABLE_HEADLESS` |
| `EAI_EXTRACTION_FAILED` | 502 | модель не вернула валидный JSON |

## Подход: LLM vs библиотеки (архитектура и развитие)

«LLM или готовые библиотеки» — ложная дихотомия. Правильный подход — **гибрид по слоям**, и сервис устроен именно так:

1. **Детерминированно из разметки (первым).** `ld+json` (schema.org/Product), OpenGraph, microdata — бесплатно, точно, без галлюцинаций. Закрывает большую долю страниц даром. Читать поля у размеченных страниц LLM **не нужен** — это переплата.
2. **LLM — генерализатор и фоллбэк.** Для страниц без нормальной разметки и для **нормализации** (свести разные сайты и разные названия характеристик в одну offer-схему **без правил под каждый сайт**). Это и есть настоящая ценность LLM здесь, а не «чтение» страницы.
3. **Headless (Playwright) — только при необходимости** (JS/SPA), кормит слои 1 и 2.
4. **Пер-сайтовые селекторы — только если 2–3 поставщика дают основной объём** (точечно поднять точность; писать на тысячи сайтов — ад поддержки).

**Готовые сервисы экстракции** (Diffbot / Zyte / Apify / Oxylabs) решают JS/антибот/extraction за деньги, но: дорого, данные уходят третьей стороне, и под РФ-контекст обычно непрактично (оплата/доступность/данные). По умолчанию — мимо.

**Фид вместо скрейпинга — для постоянных поставщиков.** Парсинг-по-ссылке хорош для разовых/длиннохвостовых товаров. Для постоянных поставщиков Quotcat надёжнее брать фид/API/прайс (коннектор каталога), а не скрести HTML. Фича делится: известный поставщик → фид; произвольная ссылка → парсер.

**Ожидания по качеству.** Парсинг товарных страниц — длиннохвостовая задача (JS, антибот, цена под логином/по региону, отсутствие разметки). Цель — «хорошо на большинстве, мягкий отказ на сложных», а не «любой URL на 100%». Перед углублением — прогнать **15–20 реальных ссылок поставщиков Quotcat** через демо и замерить % распознанных полей: это даёт честный потолок на ВАШИХ сайтах и показывает, куда вкладываться (разметка / headless / пер-сайт).

## Известные ограничения
- **Цена на JS-рендеринге** на «жирной» странице: если у страницы есть текст, но цена подставляется скриптом, `isThin` не сработает и headless не включится — цена вернётся `null`. Возможное улучшение: повтор через headless, когда `price === null` и `ENABLE_HEADLESS=1`.
- **B2B «цена по запросу»**: если цены нет на публичной странице — извлечь нечего (`price: null`), это корректное поведение.
